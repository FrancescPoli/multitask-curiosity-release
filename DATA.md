# Data availability & exclusions

This (public) repository deliberately **excludes** two classes of source data that cannot be redistributed: infant participant data (personally identifiable) and the HCP-derived structural connectomes (governed by HCP data-use terms). The exclusions are enforced by `.gitignore`. Everything needed to reproduce the analyses *downstream* of those restricted inputs is included (anonymized infant data + model-derived tables).

The full data and history are retained privately in the original `multitask-curiosity` repository.

------------------------------------------------------------------------

## Excluded (not in this repo)

### A. Infant foraging — raw data & identifiers *(PII)*

`analysis/Cognitive_analysis/Data/Infant_Data/` - `Roris_nostd_final.csv`, `Roris_smiley_final.csv`, `data_MPI.csv`, `data4R.csv` — raw per-trial records with participant IDs and unused measures (looking time, saccadic latency, lingering/dwell, etc.). - `Roris_smiley_demographics.csv` — dates of birth, infant sex, maternal age/education. - `infant_id_key.csv` — the `subj_id ↔ original-ID` re-identification key. **Never share.**

**Included instead:** `infant_foraging_release.csv` — anonymized, one row per trial, columns `dataset, subj_id, nseq, ntrialseq, ntrialsubj, event, D, I, H` only. `subj_id` is an arbitrary integer; the mapping back to real participants exists only in the private key. (The aggregate `infant_pool_summary.csv` / `infant_datasets_demographics_summary.csv` — group counts and mean ages — are retained as shareable summaries.)

> **All derived infant results are shareable.** Everything under `analysis/Cognitive_analysis/Results/` (the fitted-parameter tables, the saved `glmer`/model `.rds`, and the `infant_plots/` traces) was regenerated by running the pipeline on the anonymized release file, so it contains only `subj_id` (verified: model frames and trace filenames carry plain integers, no raw IDs). Re-running `fit_infant_mvt.R` + `plot_infant_mvt.R` reproduces them identically (subject relabeling does not change any estimate).

### B. HCP structural connectomes *(HCP data-use terms)*

`analysis/Network_analysis/Data/Atlases/Schaefer/` - `harmonized_connectomes.mat`, `age.mat` — derived from the Human Connectome Project.

**Included instead:** the public Schaefer-100 atlas files (`Schaefer100x17.txt`, `schaefer100_coords.npy`) and the *derived* per-subject topology table `Network_analysis/Results/human_topological_metrics.csv`, which lets the network analysis run without the raw imaging. (If your HCP agreement treats per-subject derived metrics as restricted, exclude that CSV too.)

## How to reproduce

Restricted **preprocessing** steps need their original inputs (obtained under the relevant agreements); everything downstream runs from the shared files.

| Step | Needs | Status in this repo |
|------------------------|------------------------|------------------------|
| `Cognitive_analysis/make_infant_release.R` | raw infant files (private) | regenerates `infant_foraging_release.csv` |
| `Cognitive_analysis/fit_infant_mvt.R` | `infant_foraging_release.csv` | **runs from the shared file** |
| `Network_analysis/extract_schaefer_metrics.py` | HCP `*.mat` (obtain from HCP) | regenerates `human_topological_metrics.csv` |
| topology / synaptic / compositional analyses | the included derived tables | **run from shared files** |

To obtain restricted inputs: infant data — contact the authors (subject to ethics/consent); HCP data — https://www.humanconnectome.org/ under the HCP Open Access / Restricted terms.
